# Printf 

Цель проекта заключалась в воссоздании упрощенной версии функции `printf` стандартной библиотеки stdio.h языка `C`.

## Реализованные спецификаторы

| Спецификатор | Описание            |
| :---:        | :---:               |
| `%%`         | Вывод одного символа `%`
| `%c`         | Вывод одного символа `char`
| `%s`         | Вывод строки
| `%d`         | Вывод числа типа `int` в десятичной системе счисления
| `%b`         | Вывод числа в двоичной системе счисления
| `%o`         | Вывод числа в восьмеричной системе счисления
| `%x`         | Вывод числа в шестнадцатеричной системе счисления

## Реализация спецификаторов

Для вывода используется буфер на 128 символов, который выводится при его заполнении, завершении функции или встрече нулевого символа.

Так как `%%` не требует передачи аргументов, при встрече символа % сначала проводится проверка на соответствие случаю `%%` и уже при отсутствиии совпадения совершается переход с помощью jump-таблицы. В случае невалидного значения выводится сообщение об ошибке. 

## Передача аргументов

Для передачи всех аргументов вначале в память сохраняется значение **RSP**, а также адрес возврата, после этого все регистры, используемые в соглашении **System V AMD64 ABI**, кладутся в стек. Затем, после завершения работы функции, регистру **RSP** возвращается его старое сохраненное значение.


### Обработка `%с`

Символ кладется в буффер, который сразу же выводится


### Обработка `%s`

Определяется длина строки, а затем адрес строки, переданный в функцию как параметр, вместе с длинной строки передается в системный вызов.

### Обработка `%b`, `%o`, `%x`

Используется общая функция вывода числа при помощи деления его на цифры битовым сдвигом и посимвольного вывода после предварительной проверки на минус в числе. Как аргументы функции переадются максимальная возможная цифра и величина сдвига.

### Обработка `%d`

Работает аналогично предыдущим трем спецификатором, за исключением того что вместо битового сдвига деление на 10.
